请基于当前仓库现有代码，优化运营监控总览页面 UI，重点处理以下几点：

1. 环境进度未满时增加周期性发光扫光动效
- 位置：系统执行态势里的应用卡片，卡片底部环境信息/环境进度条区域。
- 条件：只有环境进度小于 100% 时触发动效；100% 不需要扫光。
- 效果方向：类似“暗影燎原”那种能量光线扫过的感觉，但不要做得太夸张。
- 表现建议：
  - 在环境条或环境信息胶囊上添加一条柔和光带，从左到右周期性扫过。
  - 光带可用 cyan / emerald / amber / violet 按当前环境色继承。
  - 使用伪元素、linear-gradient、mask 或 background-position 实现，不要新增重型依赖。
  - 周期建议 3s 到 5s，每次扫过后有短暂停顿。
  - 暗色模式下光感更明显，亮色模式下要克制，不能刺眼。
  - 遵守 `prefers-reduced-motion: reduce`，用户减少动效时关闭或弱化动画。

2. 优化右侧“状态消息”列表的信息密度
- 当前问题：第一眼看到的信息太少，虽然 hover 可以看到完整内容，但默认可读内容不足。
- 目标：让每条消息第一眼能看到更多标题和内容。
- 调整建议：
  - 将 aside / 状态消息容器的 padding 改得更紧凑，例如 `8px 0`，但注意不要贴边难看。
  - 每条消息里，把标题放到第一行作为主视觉信息，例如 `ACT_合肥发布`、`PAY_生产发布`。
  - 状态标签和时间可以弱化处理：放到标题行右侧、标题下方的小字，或用左侧状态点/细色条表达。
  - 第二行展示消息摘要，让“发布复核节点节点，EXECUTE...”这类内容默认露出更多。
  - 保留 hover 展示完整内容的能力，但默认状态也要更可读。
  - 注意中英文、下划线标题、长标题的截断策略，避免布局抖动或文本重叠。

3. 额外可做的视觉与体验优化
- 状态消息列表建议进一步压缩卡片内边距和间距，让右侧栏一屏展示更多条。
- 当前 selected / latest 消息可以保留高亮边框，但不要每张卡都过度发光，避免视觉噪音。
- 右侧时间轴圆点可以和状态颜色一致，并考虑让最新消息圆点略大或有轻微 pulse。
- 环境卡片底部的小进度条可以统一高度和最小宽度，避免不同环境数量导致视觉不齐。
- 检查暗色和亮色主题下的对比度，尤其是小字、时间、进度百分比。
- 不要新增生产依赖，优先复用现有组件、CSS 变量、主题变量和样式组织方式。
- 改动保持小而集中，不要重构无关页面。
